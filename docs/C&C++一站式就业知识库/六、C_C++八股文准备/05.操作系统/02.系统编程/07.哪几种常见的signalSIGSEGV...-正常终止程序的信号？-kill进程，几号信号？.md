# 07.哪几种常见的 signal? SIGSEGV... -> 正常终止程序的信号？-> kill 进程，几号信号？


常见的信号有很多种，以下列举几种常见的：



1. SIGSEGV：表示程序访问了无效的内存地址。
2. SIGABRT：表示程序自己调用了abort()函数来终止程序。
3. SIGINT：表示程序收到了中断信号（通常是由用户按下Ctrl+C发送的）。
4. SIGTERM：表示程序收到了终止信号（通常是由kill命令发送的，默认为15号信号）。
5. SIGKILL：表示强制杀死进程（无法被阻塞、处理或忽略），一般用于紧急情况。它会立即终止进程，不管进程是否正在执行任务。
6. SIGPIPE：表示向已关闭的管道或Socket写数据时，系统会发送这个信号给进程，以提示进程该操作已失败。



正常终止程序可以通过exit()函数来实现，也可以使用return语句从main函数返回0来实现。kill进程使用kill命令，在Linux系统中默认发送15号信号(SIGTERM)。
